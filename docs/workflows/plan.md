# 计划阶段工作流 (Plan Phase Workflow)

## 目标

将研究阶段的分析转化为可执行的实施计划。

## 步骤

1. **检索/创建文档**: 如果相关文档不存在，创建一个新的 Markdown 文件
2. **更新设计**: 在写代码前，先用**中文**在文档中描述变更
3. **用户确认**: (可选) 如果变动巨大，向用户展示文档变更摘要

## 输出

计划阶段产出物存放于 `.artifacts/plan/`，包含：
- 实施计划文档（中文）
- 具体改动范围
- 风险评估

## 原则

- 计划必须可被"初级开发人员直接执行"
- 每一步都有明确的验证标准
- 标注假设和依赖
