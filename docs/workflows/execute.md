# 执行阶段工作流 (Execute Phase Workflow)

## 目标

严格按照计划实施，避免计划外修改。

## 步骤

1. **按计划执行**: 严格按照 `.artifacts/plan/` 中的计划实施
2. **分阶段提交**: 每完成一个有意义的小单元进行提交
3. **保持同步**: 代码必须严格符合文档中的定义

## 执行日志

执行日志存放于 `.artifacts/execute/`，记录：
- 实际执行步骤
- 与计划的偏差（如有）
- 验证结果

## 变更守卫

- 执行前运行 `go build ./...` 确保编译通过
- 执行后运行 `go vet ./...` 检查常见问题
- 遵循 `engine.Builder` 签名规范

## 原则

- **严格按计划**: 不在执行阶段引入计划外的改动
- **发现偏差立即记录**: 如有与计划不符之处，记录到执行日志
- **小步提交**: 避免大量一次性提交
